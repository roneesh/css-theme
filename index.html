Spacing

These styles use a base rhythm of 10px. Various parts of the app will use an empty div with a .vr_* class applied, like so:

<div class="vr_1"></div> <-- 10px of spacing below -->
<div class="vr_2"></div> <-- 20px of spacing below -->
<div class="vr_3"></div> <-- 30px of spacing below -->
<div class="vr_4"></div> <-- 40px of spacing below -->
...up to 9 vr's.

In some cases, elements or modules will include the base rhythm in their styling, but as a general rule elements and modules should avoid setting their own spacing in the app to avoid CSS side effects.

Text

All apps use text, these are p tags and span tags. Your app should consistently define sizes for various levels of text.

Standard Text:

Lorem ipsum dolor sit amet

Code: <p>Lorem ipsum dolor sit amet</p>

Small Text:

Lorem ipsum dolor sit amet

<p class="txt-small">Lorem ipsum dolor sit amet</p>

Large Text:

Lorem ipsum dolor sit amet

<p class="txt-large">Lorem ipsum dolor sit amet</p>

Centered Text:

Lorem ipsum dolor sit amet

<p class="txt-centered">Lorem ipsum dolor sit amet</p>

Bold and Italic Text:

Bold, Italic

<p class="txt-centered">Lorem ipsum dolor sit amet</p>

Links

All apps use links, these are a tags. Your app should not only define styles for links, but also for the three pseudo classes of links: :hover, :focus, :active

Standard link:

Back to top link! <a href="#">Lorem ipsum dolor sit amet</a>

Link in focus, hover or active (all same styling):

Back to top link!

Lists

Most every app uses lists. These are <ul> and <ol> tags.

Vertical List:

  • Foo
  • Bar
  • Buz
<ul class="v-list">
   <li class="v-list__item">Foo</li>
   <li class="v-list__item">Bar</li>
   <li class="v-list__item">Buz</li>
</ul>

Horizontal List:

  • Foo
  • Bar
  • Buz
<ul class="h-list">
   <li class="h-list__item">Foo</li>
   <li class="h-list__item">Bar</li>
   <li class="h-list__item">Buz</li>
</ul>

Note that horizontal lists have no list style type.

Ordered Lists:

  1. Foo
  2. Bar
  3. Buz
  1. Foo
  2. Bar
  3. Buz
<ol class="h-list">
   <li class="h-list__item">Foo</li>
   <li class="h-list__item">Bar</li>
   <li class="h-list__item">Buz</li>
</ol>

Ordered lists will have their number in the list in both horizontal and veritcal configurations.

Buttons

Every app uses buttons, and generally there are multiple styles.

Generally, buttons are around 34px high, so they can be easily pressed by both a mouse, or a users finger on a mobile device.

<button class="btn">
   Button
</button>

Danger Button:

<button class="btn btn--danger">
   Button
</button>

Cancel Button:

<button class="btn btn--cancel">
   Button
</button>

Finally, it's a common pattern to want a button to look like a link. However it's important to use a tags when a user will leave the page, and use the button tag when the press will trigger an effect on the current page.

Link Button:

<button class="btn btn--link">
   I'm actually a button...
</button>

Tables

Occasionally apps use tables, and they're normally hard to get right. This is a simple, clear table treatment with a responsive design. The responsive design simply collapses rows into blocks, showing each row as a block item. The table utilizes a fixed table layout, with text overflow clipped by ellipses. You can add hover states by adding table--hover.

# Name email Verified? Opted In? On Hold?
1 John Smith John@Smith.com Yes No No
2 Jane Smith Jane@Smith.com Yes No No
