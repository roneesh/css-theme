A Horizontally Responsive Grid

CSS grid is similar to flexbox, it is a parent -> child relationship between html elements. A single element is the "parent" container, it has a property of "display: grid", and then each direct child ("> *") is placed in the grid. Children of children are not considered to be "in the grid", nor are elements adjacent to or above the parent.

This very basic grid that defines no rows, only columns. So browsers will simply define rows per their default algorithm. This means most likely rows will be defined to fit the height of the content. The columns are in relative units of "1fr", fr meaning "fraction". if every column is a width of "1 fraction", then all cols are equal width.

Because this grid uses a 1fr width, it's horizontally responsive. It will fit in its container as many columns as you ask, all equal sized.

<div class="grid">
   <img .../>
   <img .../>
   <img .../>
   <img .../>
</div>

A Vertically Responsive Grid

Using rows and the various units of height they can have, we can define a grid that fits its column into the viewport both vertically as well as horizontally.

Traditional "Responsive" design simply made sure things fit the "width" of the viewport, and really made no regards to the height. It was assumed users would scroll.

With CSS grid, we can make a layout that responds to vertically shrinking/increasing the height of the viewport.

In this example, my headshot is tertiary to the nav links, so my headshot shrinks as you resize the page, and the links always remain in the viewport. Try it out-- shrink or grow the page in any direction and see if you can remove the links from view.

A Mondrian Using CSS Grid

The above two grids utilized only columns or rows. They were an example of using CSS Grid in a single dimension.

A "Mondrian" is a grid layout of blocks of color, just like the paintings of Piet Mondrian. A Mondrian is inherently a 2-D layout, so it's ideal to use CSS Grid to recreate this.

<ul class="mondrian">
   <li><img .../></li>
   <li><img .../></li>
   <li><img .../></li>
   <li><img .../></li>
</ul>

A Layout Using: Flow, Float, Flexbox, Grid

It might be hard to see how the various CSS layout schemes can all be composed together to create layouts that are great for users. Luckily there's a trick: think small to large.

The smallest scale of layout is between two html elements, for exmaple an "img" tag followed by a "p" tag. These elements should utilize flow to be organized:

<img alt="A kitten!">
<p>This kitten is cute!</p>

This kitten is cute!

Flow is great, it's natively mobile friendly, but for larger viewports, we might want to have some wrapping. We can achieve wrapping of text around images with "float". Sometimes float is just as simple as adding "float: left;" on the image, but it's common to "clear" your floats so the container encompasses their height.

This kitten is cute! Also this is more text about the image, so it's good if we start wrapping text around the image. We're using float, which is a really old method of layout, but, we're using float as it was intended, so it's still valid and correct to use!

There is one thing to consider though: we had to clearfix this float. Remember, if you use floats, you need to have your containers clear them.

<div class="clearfix">
   <img class="float-left" alt="A kitten!" style="padding-right: 1rem;">
   <p>This kitten is cute...</p>
</div>

Float got us a style we like, where text wraps an image, but often this specific mingling of images and text is only part of a larger layout. Now let's put this in the context of an adoption card, the card layout uses flexbox:

Adopt Fritz! Talk about a cute cat! Their hobbies include: yarn, humans, scratches and of course... jumpin' 'round!

This card is great, but wouldn't a grid of cards be better? Let's see an example in the below blue ribbon. Resize the page horizontally to see that the cards respond perfectly in a very simple .tile-grid container. This grid of cards has it all:

  • Flow + Float for the card's image and description
  • Flexbox for the entire card's vertical layout.
  • CSS Grid to layout the three cards horizontally, but also with very sensible responsive design without media queries!
