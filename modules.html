Sliders

It's a fact of life, people love when elements "woosh" around the page. So a theme that can have elements slide in and out of view would be great. This theme uses a very lightweight implementation, it utilizes a few technologies:

  1. CSS Transforms and Animations
  2. The IntersectionObserver API

I'll slide in from the left when I enter the viewport.

<div class="section js-slider slide-left" data-slide-class="slide-in">
   <p>I slide in from the left</p>
</div>

I'll slide in from the right when I enter the viewport.

<div class="section js-slider slide-right" data-slide-class="slide-in">
   <p>I slide in from the right</p>
</div>

Modals

The Modal pattern is here to stay, as annoying as I personally find them.

If you intend to implement a modal from scratch for your project, keep in mind that a modal actually has a fair amount of requirements if you want it to be semantic ad accessible.

A modal should functionally :

  1. Be openable via the click of one or more buttons.
  2. Be be programmatically openable and closable.
  3. Be able to co-exist with other modals (e.g., multiple modals should be able to function on the same page).
  4. Be able to contain multiple content types (e.g., text, images, headings, etc.).
  5. Close when the escape key is pressed.
  6. Close when clicking on the modal “backdrop”.

That isn't all, to maintain top notch accessibility, a modal should also:

  1. Should identify itself as a dialog.
  2. The modal should be properly labeled.
  3. When opened, focus should be redirected to the modal.
  4. When closed, focus should return to the activation point (i.e., the button that was clicked to open it).
  5. While the modal is open, the modal should retain focus.
  6. While the modal is closed, the modal and its contents/children should not be discoverable or focusable.
